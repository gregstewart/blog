A set of suggestions for dealing with mass mailouts and other email related applications

  1. build in mechanism to verify that the scehduled mail out task has occured and executed properly (expected numbers vs actual numbers and log each entry)
  2. consider mail server performance and know how it handles messages it can't send immediately (i.e. if the domain can't be looked up does it queue the message and send another or does it hold them up while waiting to try again, how long is the lag in processing when an incident occurs)
  3. gage the number of messages to be sent
  4. stager mailouts
  5. Presentation in CF do this: <cfset application.mailfrom = """Your name"" "> and Johnniw Walker will appear in the sent from field in the received email message
  6. Below a suggestion for PHP in the same vane, just a little more elaborate as it provides a set of headers as well to be passed by the mail message. This will stop your message from bouncing from services such as Hotmail.
    $myname = "Info";
    $myemail = "info@teacupinastorm.com";

    $contactname = "Greg Stewart";
    $contactemail = "gregs@teacupinastorm.com";

    $message = $msg;
    $subject = "Invoice process run";

    $headers .= "MIME-Version: 1.0\r\n";
    $headers .= "Content-type: text/html; charset=iso-8859-1\r\n";
    $headers .= "From: ".$myname." <".$myemail.">\r\n";
    $headers .= "To: ".$contactname." <".$contactemail.">\r\n";
    $headers .= "Reply-To: ".$myname." <".$myemail.">\r\n";
    $headers .= "X-Priority: 1\r\n";
    $headers .= "X-MSMail-Priority: High\r\n";
    $headers .= "X-Mailer: Just My Server";

    mail($contactemail, $subject, $message, $headers);
