Mark Mandel put up a post and some code for a ColdFusion function that he wrote which uses the underlying Java architecture to do some XSL transformations using the Xalan engine and thus by passing the built-in xmltransform() function that doesn't support xsl:param and xsl:import. Might come in handy for future reference. [Massimo Foti put up something similar a while back.]
