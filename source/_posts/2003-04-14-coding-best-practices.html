Use switch statements where ever possible, because if/else needs to evaluate each condition (i.e. each if statement), where as with switch statements no evaluation is required.

Reverse loops can be twice as fast as normal loops. So instead of counting up (i++), consider counting down (i--)

Also should you be doing any kind of calculation in your loops, consider moving it out of the loop. So instead of this:

for (i=0;i d=Math.sqrt(y);
j+=i*d;
}

Do this:

d=Math.sqrt(y);
for (i=0;i j+=i*d;
}

Two ways to improve your database security. If you are using ColdFusion validate your input into SQL queries by way of the <cfqueryparam> paramater for <cfquery>. This helps validate the data to a strict expected type and will abort if the supplied information doesn't match. Another way and possibly better is to move this kind of checking to the database. Set up different user types that have different access privileges and hence can't execute statements like drop if say display queries are run based on URL IDs. This is pretty secure and also moves the processing to the database rather than burdening the application server further.
